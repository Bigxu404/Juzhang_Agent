# Warm Gathering（黄昏胡桃木）官网完整设计方案说明

版本：v0.2（方案级）  
适用：官网 Web（Landing / World / Capabilities / Trust / Download）+ 未来 App 视觉一致性

> 产品定位（不讨论功能细节）：**朋友式 Agent**（关系先于功能），但不拟人表演。  
> 视觉氛围：**黄昏暖灯 + 深胡桃木桌面**。  
> 叙事重点：**积累与变化**（不用时间轴）。  
> 能力要求：**系统能力外显**，但呈现克制，不做控制台。

---

## 1. 设计北极星（North Star）

### 1.1 必须被“看见”的三件事
1) **关系空间**：像“屋里留着灯”，你随时能回来。  
2) **会积累的存在**：成长表现为“厚度与结构”，不是时间节点。  
3) **Agent 可用性**：能力外显、可调用、可控边界。

### 1.2 为什么是黄昏 + 胡桃木
- **黄昏暖灯**：安全、放松、可停留。
- **深胡桃木**：成熟、长期主义的现实空间，但只做氛围层，避免家装/餐饮风。
- **低饱和**：耐看、可信。

---

## 2. 视觉母题（Visual Motif）：桌面叠放

> 不用时间轴表达成长，改用“桌面上碎片的叠放与结构化”。

### 2.1 母题组件（全站复用）
- **暖灯光晕（Lamp Glow）**：Hero 背景径向渐变（品牌记忆点）。
- **桌面便签（Fragments）**：世界观碎片卡（桌上的纸条）。
- **叠放卡片（Layers）**：成长厚度（习惯/理解/片段/边界）。
- **杯垫圆环（Coaster Ring）**：边界与私密范围（极淡压纹）。
- **细颗粒噪点（Noise）**：雾面质感（2–3% opacity）。

### 2.2 设计含义
- 用“空间与物的组织方式”建立朋友感，而不是人物与表演。
- 这些元素可系统化复用：主视觉、卡片、空状态、动效、icon。

---

## 3. 颜色系统（Design Tokens）

原则：**暖色承载关系**，**冷静补色承载系统能力**。

### 3.1 Light（Dusk）核心 Token（摘要）
- `bg-0 #FBF6EF` / `bg-1 #FFFFFF` / `bg-2 #F3E9DD`
- `text-1 #201A16` / `text-2 #5C524A` / `text-3 #A79B91`
- `line-1 #E6D7C8`
- `brand-1 #B04E12` / `brand-2 #C35E1C`
- `cool-1 #2F6F5E`
- 胡桃木装饰：`wood-1 #6B442C`（仅背景/装饰）

### 3.2 Dark（Dusk）核心 Token（摘要）
- `bg-0 #100B08` / `bg-1 #18110D` / `bg-2 #221813`
- `text-1 #F6EFE6`
- `brand-1 #E39B62`
- `cool-1 #75B6A6`

### 3.3 暗色模式策略
- 默认：Auto（跟随 `prefers-color-scheme`）
- 用户可切：Auto / Light / Dark（手动优先系统）

---

## 4. 字体与排版（Typography）

### 4.1 字体
- 中文：PingFang SC（优先）/ Noto Sans SC（fallback）
- 英文数字：Inter

### 4.2 字号体系（建议）
- Hero Title：46/56，-0.02em
- Hero Sub：18/30
- H1：32/42；H2：24/34；H3：18/28
- Body：16/28；Body-sm：14/24；Caption：12/18（+0.02em）

### 4.3 行宽规则
- 正文最大行宽：约 22–28 个中文/行（避免满屏）

---
## 5. 栅格与间距（Layout & Spacing）

- 最大内容宽：1120px
- 页面边距：桌面 80px / 平板 40px / 移动 20px
- 12 栏栅格，gutter 24px

间距（只用）：4/8/12/16/24/32/48/72/96/128
- Hero → 下一模块：96
- 模块上下：72
- 卡片组间距：24
- 卡片内边距：24（移动 20）

---

## 6. 形状、阴影、纹理（Material）

- 圆角：输入/标签 10；卡片 18；大容器 26
- 阴影（Light）：卡片 `0 10 30 rgba(32,26,22,0.10)`；hover 上移 2px + 阴影加强
- 纹理：
  - 全站 noise 2–3%（必须）
  - Hero 暖光晕（必须）
  - 蒸汽线条（可选，opacity 0.05，2–3 根以内）

---

## 7. 动效规范（Motion）

动效是“呼吸”，不是表演。

- Hover：160ms，translateY(-2px)
- Modal/Drawer：240ms
- Hero 背景光晕：1200–2400ms 极轻微呼吸（透明度/半径微变）
- 禁止：大幅位移、弹跳、粒子爆炸、强旋转

---

## 8. 组件系统（关键组件 + 规则）

### 8.1 Button
- Primary：44px 高，radius 12，bg brand-1，hover brand-2
- Secondary：描边 line-1，hover surface-warm
- Disabled：降低对比，不做花哨

### 8.2 Fragment Card（世界碎片便签，品牌核心组件）
- radius 18，padding 24
- 标题 18/28（≤12字）+ 正文 14/24（≤2行）
- 角标：6px 暖点（brand-1 + blur）
- 在首页/World 顶部可允许轻微错位叠放（±2°、2–6px）；正文阅读区禁止

### 8.3 Layer Card（成长叠卡）
- 3–5 张叠放，偏移 10px
- 只让最上层承载文本，其余露边缘与阴影

### 8.4 Capability Card（能力外显）
- 图标统一用 cool-1（理性、系统感）
- 激活态用 2px 竖条 brand-1（克制、可识别）
- 卡片内“边界句”固定存在（例如：不替你做决定 / 你随时可叫停）

### 8.5 Theme Toggle
- Header 右侧：Auto / Light / Dark
- 默认显示 Auto（低存在感），下拉宽 160px、行高 40px

---
## 9. 信息架构（IA）与页面方案

导航（Header）：**世界 / 能力 / 信任 / 下载** + 主题切换

### 9.1 Landing（首页）模块顺序（固定）
1. **Hero：黄昏空间**（暖灯光晕 + 胡桃木桌面意象）
2. **你得到的是什么（3 卡）**：在场 / 积累 / 边界
3. **关系如何变厚（叠卡）**：形成习惯 / 积累理解 / 留下片段 / 保持边界
4. **世界碎片预览**：Fragments 卡片墙（2×3）
5. **能力外显预览**：Capabilities 卡片墙（2×3）
6. **Trust 契约**：记忆可控 / 边界清晰 / 随时可迁移
7. Footer：隐私与边界 / 条款 / 联系我们

**关键说明**：
- 首页前半段先建立“空间与关系”，后半段再呈现能力；避免一上来像工具页。

### 9.2 World（世界页）
目标：让用户相信世界观来自“片段与结构”，不是剧情设定。
- 页标题 + 两句内副文案
- 分组（按生活域，不按时间）：桌面与片段 / 日常结构 / 偏好与习惯 / 互动方式 / 边界与规则
- 支持搜索与筛选（可选）：全部 / 在场 / 记忆 / 边界 / 行动

### 9.3 Capabilities（能力页）
目标：能力外显，但语气必须“协作 + 边界”。
- 分组建议：信息与理解 / 写作与表达 / 计划与推进 / 记忆与整理 / 边界与提醒
- 每个能力卡：做什么 + 不会做什么（边界句）+ CTA

### 9.4 Trust（信任页）
目标：把“朋友关系”落到可验证的控制权上。
- 三块卡片：记忆可控 / 边界清晰 / 随时可迁移
- 解释用短句，不写长法律腔；法律条款可在页脚链接。

### 9.5 Download / Waitlist（下载/等候页）
目标：转化，但不营销。
- 标题 + 一句克制副文案
- 表单：邮箱 / 平台 / 优先体验方向

## 10. 文案系统（与碎片卡库的结合方式）

### 10.1 全站文案来源
- 页面主叙事（Hero、模块标题、契约）来自 **B1 Copy System**
- 世界观与长期运营内容来自 **碎片卡库**（Fragments Library）

### 10.2 碎片卡库在官网的落点
- Landing：展示 6 张（2×3）作为“世界一角”
- World：完整展示 + 分组 + 搜索
- 空状态：从碎片卡库选取更克制的句子做提示

### 10.3 写作约束（必须遵守）
- 每屏意象词≤1（Hero≤2）
- 禁用：永远/一定/绝不、懂你、治愈、技术炫耀

---

## 11. 视觉资产清单（交付给设计/前端）

### 11.1 必备资产
- Logo（横版 + icon-only）
- App icon（iOS/Android）
- 线性图标 12 个（统一线宽 1.8px，圆角端点）
- 背景 Noise 纹理（1 张透明 PNG/SVG，可平铺）
- 杯垫圆环压纹（SVG）
- 蒸汽线条（SVG，2–3 条）

### 11.2 主视觉（Hero）要素
- 暖灯光晕（径向渐变）
- 深胡桃木淡纹理（只在 Hero/页头出现）
- 叠放的卡片（Fragments + Capability）

---

## 12. 可访问性与可用性（最低要求）
- 文本对比度：正文与背景至少满足 WCAG AA（尤其暗色模式）
- 焦点态（Focus）：所有可交互元素必须有清晰 focus ring（建议用 brand-3 背景 + 1px brand-1 边）
- 主题切换：键盘可操作；切换后不跳动布局
