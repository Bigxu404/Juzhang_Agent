# PRD 03: iOS Native Infrastructure (原生躯体底座)

## 1. 产品定位与目标
本模块是Agent在物理世界的“感官”与“躯体”。
在移动端做Agent，核心优势不是算力，而是**无缝集成到用户的生活流中（Share Extension）、随时随地（随时唤醒）以及拟真的交互体验（活人感UI）**。我们将完全基于 Swift / SwiftUI 打造一个极具温度的原生客户端。

## 2. 核心功能设计

### 2.1 Share Extension (分享拦截与灵感收集)
**阅读与灵感记录的核心入口。**
用户在微信阅读公众号长文、在淘宝刷到好物、在小红书看到攻略时，不需要繁琐地复制链接再打开App，而是直接通过系统原生的分享菜单将内容发送给Agent。

- **输入类型**：URL链接、纯文本、图片（截图）。
- **处理流程**：
  1. 用户点击系统分享按钮，选择我们的产品（如“老友记”）。
  2. 弹出一个极简的交互面板（Mini Chat UI）。
  3. 用户可以附加一句话：“帮我总结一下核心观点”，或者不说话直接发送。
  4. Share Extension 在后台将数据发送给后端的 `Agent Core Loop`。
  5. 服务端触发后台的 `Explore Subagent` 去抓取网页并提取关键信息。
  6. 客户端在稍后（或实时）通过推送或主界面更新，收到Agent的吐槽或总结：“这篇文章太长了，不过我帮你提炼了三点……已经存到Notion里了。”

### 2.2 Background Tasks & Push Notifications (后台唤醒与主动性)
突破传统App“你打开我才理你”的被动设定。结合PRD 02中的价值观更新系统，让Agent“活”起来。

- **Background Fetch**：系统定期在后台唤醒App，App向服务端拉取是否有新的“冲浪结果”或“思考任务”。
- **Interactive Push Notifications (互动式推送)**：
  - 当Agent在后台帮用户完成了某项长期任务（例如比价结束、阅读摘要生成），通过 APNs 推送结果。
  - 推送不仅是干巴巴的文字，而是**带有交互按钮**的通知（例如：[存入清单]、[听你细说]、[不太感兴趣]），用户在锁屏界面即可完成指令闭环。

### 2.3 “活人感”的原生UI/UX设计
交互细节决定了产品是“工具”还是“朋友”。
- **流式输出与打字机特效**：WebSocket 实时接收文字，匹配自然的打字速度，偶尔模拟“停顿思考”的卡顿感。
- **Haptic Feedback (触觉反馈)**：在消息发送、Agent开始思考、任务完成存入MCP时，触发不同强度的震动，增强物理世界的存在感。
- **状态指示器 (State Indicators)**：不再是一个冰冷的Loading圈。借鉴Claude Code，当主Agent派发子任务时，UI上应显式展示其思考过程，例如：“🧐 正在全网搜索这款鞋的价格...”、“📝 正在写入你的Notion...”，让用户感知到其正在“努力干活”。

### 2.4 App Intents & Siri 集成 (未来扩展)
- 允许用户通过喊“Hey Siri，跟老友说我今天好累”，直接通过语音或快捷指令唤醒Agent的陪伴模式。

## 3. MVP 阶段的取舍
- **排除**：Voice Mode (实时语音通话) 暂时排除，集中精力打磨文本和图文多模态交互体验。
- **保留**：Share Extension 是重中之重，必须在第一阶段跑通，这是高频触发产品价值的命脉。
