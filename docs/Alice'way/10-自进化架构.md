# 第十章：自进化架构

真正的自进化，要能写代码、要能部署、要能撤销。

---

## 三层自进化架构

- **L0：人格反思**：修改 `[MUTABLE]` 分区，低频触发。
- **L1：配置热更新**：接收服务端下发配置。
- **L2：代码级自进化**：
  - **L2a**：内置工具参数白名单。
  - **L2b**：Widget 小组件。
  - **L2c**：自定义页面（沙盒 WebView）。

---

## 代码生成管线

1. 构建代码生成 Prompt（含 API 契约和沙盒约束）。
2. LLM 生成 XML 格式代码。
3. **SecurityScanner** 静态安全扫描（禁止 eval、外链、跨域等）。
4. 通过 **AliceBridge**（白名单 IPC）在沙箱中运行。

---

## 安全与门控

- **[PROTECTED] 区**：核心身份定义，代码级强制限制，AI 和用户均不可修改。
- **会话级门控**：强制要求先调用 `read_system_map` 才能使用 L2 工具。
- **主动提案**：`propose_evolution` 不直接执行，需用户确认卡片。

---

## 撤销机制

执行 L2 操作前自动执行 `pushUndo` 快照。支持通过 UI 撤销历史回退到任意版本，快照持久化到磁盘。

---

## 核心原则

AI 的自主性永远服从于用户的可控性。自进化的本质是"在用户允许的范围内，AI 帮助系统变得更好"。
