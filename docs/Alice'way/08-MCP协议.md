# 第八章：MCP 协议

如果 Agent 的能力完全内置，它就是一个封闭的系统。

---

## 内置工具的局限

内置工具是基础，但无法覆盖所有外部系统（WPS、内部数据库、Slack 等）。
**MCP (Model Context Protocol)** 让 Agent 可以在运行时动态发现和调用外部能力，而不需要修改 Agent 本身。

---

## MCP 的工作模型

MCP 工具在 Agent 视角下与内置工具一致，通过名称前缀 `mcp_{serverId}_{toolName}` 进行路由。

---

## 关键设计细节

1. **懒连接**：注册时不建立连接，第一次调用工具时才连接，不阻塞启动。
2. **工具描述截断**：对过长的工具描述做长度限制（约 2048 字符），防止填满上下文。
3. **超时保护**：调用超时（约 30 秒）则报错返回。
4. **断线重连**：监听连接断开事件并自动尝试重连。
5. **权限管控**：默认最保守设置（非只读、需确认、非并发安全），支持通过规则显式放行。

---

## 工具发现的质量问题

工具过多会降低模型 Tool Choice 准确率。解决方向：
- 高质量的 `when_to_use` 描述。
- 按场景动态加载工具。
- 使用 ToolSearch 工具让模型先搜索再决定。
