# 第十三章：可观测性

不可观测的系统等于黑盒；黑盒不能优化，也不能被信任。

---

## Agent 的可观测性挑战

- **非线性执行**：多轮迭代、递归子任务。
- **时间跨度大**：任务持续时间长，需记录中间过程。
- **原因难以归咎**：需区分是工具、上下文还是权限问题。

---

## OTel 三信号应用

1. **Traces（追踪）**：记录完整调用链。需区分 `blocked_on_user`（等待确认）和 `execution`（实际执行）时间。
2. **Metrics（指标）**：Token 消耗、工具调用频率、压缩触发次数。
3. **Logs（结构化日志）**：关键事件记录，按日期落盘。

---

## LLM 调用分类 (caller)

每次调用需标记来源：`main`（主对话）、`memory`（记忆提取）、`compact`（压缩）、`classifier`（权限）、`subagent`（子 Agent）。

---

## 隐私与安全

- **隐私保护默认**：不记录 Prompt 内容和工具参数，只记录元数据。
- **详细日志模式**：需用户显式开启用于调试。

---

## 多 Agent 可观测性

每个 Agent 实例有唯一 ID，所有日志携带该 ID。在 Trace 中形成树状调用链。
