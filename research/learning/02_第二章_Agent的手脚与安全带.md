# 第二章：Agent 的手脚与安全带（工具调用与 Harness 拦截）

本章我们探讨了 Agent 在拥有强大大脑后，如何安全地与外部世界交互。我们从“爱马仕 (Harness)”的行业黑话切入，深入解析了工具权限分级与安全拦截机制。

## 1. Harness（爱马仕/脚手架）的演进与本质

### Q：Harness（爱马仕）到底是什么？模型层面的 Harness 和我们系统里的 Harness 有什么区别？
**A：** 
*   **名字的由来**：`Harness` 本意是“马具”，引申为控制大模型这匹“野马”的外围代码（脚手架）。国内 AI 圈借用做高级马具起家的奢侈品牌“爱马仕”来戏称牛逼的 Agent Harness。
*   **行业的演进**：
    *   **过去（代码级 Harness）**：模型很笨，开发者要写极其厚重复杂的外部代码（如 LangChain/AutoGPT）来教模型循环、纠错、调工具，Bug 频发（马具容易断）。
    *   **现在（模型级 Harness / Native Agentic）**：像 MiniMax M2.7 等最前沿模型，在出厂训练时就喂了海量工具调用轨迹数据。模型**天生自带马具感**，原生就知道如何组合工具、如何自我纠错，甚至能反过来修改脚手架代码。
*   **我们的架构定位**：既然最顶级的模型已经把复杂的“马具”长在了自己身上，我们在架构设计中就把大量繁杂步骤交给模型的“内生能力”，而把我们手写的 Harness 极度克制地精简为一件事：**Level 3 权限的绝对拦截（最后一道安全防线）**。

## 2. 权限分级与拦截机制

### 核心机制：工具的安全等级标签
在 `ToolDispatcher` 中，每个工具必须明确安全等级：
*   **Level 1（绿色 - 静默执行）**：纯只读操作（如 `WebSearch`），Agent 在循环中随意调用，无需打扰用户。
*   **Level 2（黄色 - 事后知会）**：轻微副作用操作（如存草稿），Agent 先斩后奏，边做边在流式输出中告知用户。
*   **Level 3（红色 - 强制拦截）**：高危写操作（如发邮件、删数据、付款），触发“时间静止”魔法。

### “时间静止”魔法的工作流
1.  **挂起（Suspend）**：调用高危代码被冻结，Agent 思考暂停。
2.  **发信号（Event）**：后端通过 WebSocket 向前端发送 `PERMISSION_REQ` 信号（包含操作意图和参数）。
3.  **等待审判（Awaiting）**：后端挂起等待，直到前端发回 `ALLOW`（同意）才解冻执行，或 `DENY`（拒绝）则报错并由 Agent 道歉退回。如果连续拒绝 3 次，系统触发防骚扰熔断机制。

## 3. PM 视角的权限拦截卡片 (Permission UI) 设计

关于高危拦截弹窗的产品交互设计：
1.  **一致性与强制性**：风格应与“意图澄清卡片”保持一致，但要突出强制决策的属性——**不决策就走不下去，不提供跳过按钮**。
2.  **信息层级与后果展示**：采用“主标题 + 次要内容”的结构，清晰地告诉用户“Agent 要干什么”以及“后果是什么”，降低用户的认知负荷和恐惧感。
3.  **纠正路径的留白**：如果用户不同意当前拟定操作（如觉得邮件语气太重），除了简单的“拒绝”，还可以考虑提供“直接修改参数”或“语音补充一句修改意见（打回重做）”的快捷通道（待后续细化）。

**总结**：无论 AI 多么聪明自主，最高权限的“扳机”永远捏在用户手里。这种克制与放权的平衡，是建立人机信任协同（Human-in-the-loop）的核心基石。
